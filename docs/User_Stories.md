# User Stories

## Story 1
As a guest user, I want to select a language when launching the app, so I can understand the interface.

Acceptance Criteria:
- Language selection is available on first launch.
- All interface text updates according to the selected language.

## Story 2
As a tourist, I want to see attractions on an interactive map, so I can easily explore nearby places.

Acceptance Criteria:
- The map loads successfully.
- Place markers are visible on the map.

## Story 3
As a user, I want to view detailed information about a place, so I can learn more about it.

Acceptance Criteria:
- Place description and images are displayed correctly.
- Data loads without errors.

## Story 4
As a registered user, I want to build a route between locations, so I can plan my trip.

Acceptance Criteria:
- Route is calculated using the routing service.
- Route is clearly displayed on the map.

## Story 5
As a traveler, I want to check the weather for a place, so I can prepare for the visit.

Acceptance Criteria:
- Current weather data is shown.
- Weather updates when a new place is selected.

## Story 6
As a user, I want to register and log in, so I can access personalized features.

Acceptance Criteria:
- User can register with email and password.
- User can log in and log out successfully.

## Story 7
As a registered user, I want to edit my profile, so I can keep my information up to date.

Acceptance Criteria:
- Profile changes are saved.
- Updated information is displayed correctly.

## Story 8
As a user, I want the app to remember my settings, so I do not need to configure them again.

Acceptance Criteria:
- Settings persist after app restart.
- Preferences load automatically on launch.
